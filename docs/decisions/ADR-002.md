# ADR-002: Mobile Technology Selection

| Field        | Value                  |
| ------------ | ---------------------- |
| **Status**   | Accepted               |
| **Author**   | @vojtechsanda          |
| **Date**     | 2026-03-26             |
| **Decision** | React Native with Expo |

## 1. Context

We are building a mobile application that helps users limit impulse purchases by converting prices to work hours, suggesting alternative uses for the money, tracking savings, and nudging the user with delayed notifications. The app includes a home-screen widget, camera/image input, push notifications, and personalization features.

### Constraints

- **Timeline:** ~1-2 months to a working product.
- **Platform priority:** Android primary, iOS nice-to-have.
- **Backend:** Supabase (PostgreSQL + Auth + Realtime + Storage). ADR-001
- **Widget:** Informational home-screen widget (nice-to-have, simple).
- **UX goal:** Playful, animated interface with celebration effects, micro-interactions, and smooth transitions.

## 2. Considered Options

### 2.1 Kotlin Multiplatform (KMP)

Briefly considered and dismissed:

- **No proper knowledge of best practices.**
- The least mature ecosystem.
- No useful added value compared to React Native.

### 2.2 Flutter (Dart)

**Pros:**

- Best animation system of all options.
- Growing ecosystem, strong widget toolkit.
- @vojtechsanda has one semester of proper Flutter experience.

**Cons:**

- **Only @vojtechsanda has any Dart/Flutter experience.** Others would need to learn Dart from scratch - a new language, new paradigms, new tooling.
- Flutter home-screen widgets still require writing native platform code (Kotlin/Swift).

**Verdict:** Objectively the strongest framework for animations and cross-platform, but the team composition makes it a poor fit given the timeline.

### 2.3 React Native with Expo (Managed Workflow) ✅ Selected

**Pros:**

- **Maximizes existing team skills.** @vojtechsanda has 3 years of React + TypeScript — roughly 80% of that knowledge transfers directly. @minotavi11 's Vue.js experience shares the same mental model, quite easy to transition. @EmaJasekova has some basic FE web knowledge which is directly applicable.
- **Expo abstracts native complexity.** Push notifications (`expo-notifications`), camera (`expo-camera`), image picker (`expo-image-picker`) are available as simple JS APIs — no native Android/iOS code needed.
- **Expo EAS** provides cloud builds — no Mac required for iOS builds.
- **Animation ecosystem is strong.** `react-native-reanimated` (60fps UI-thread animations), `lottie-react-native` (After Effects JSON animations for celebratory effects, micro-interactions), and `moti` (declarative API similar to Framer Motion) cover all requirements for a playful, animated UI.
- **Largest community and ecosystem** of all cross-platform options — most problems have been solved and documented.

**Cons:**

- Android home-screen widget support requires `react-native-android-widget` — functional but less flexible.
- Performance ceiling is lower than native for extremely graphics-heavy apps (not relevant for this project).
- The existing Kotlin template (architecture, login) cannot be reused — this work must be redone.

**Verdict:** Best fit for the team, timeline, and project requirements. Maximizes skill reuse, minimizes onboarding, and provides a clear path to both platforms.

## 3. Decision

**We choose React Native with Expo.**

The decision is driven primarily by team skill alignment and the 2-month timeline. React Native with Expo is the only option where we can build high-quality software, while everyone is productive from the beginning.

## 4. Risks and Mitigation

### R1: Widget implementation complexity

|                |                                                                                        |
| -------------- | -------------------------------------------------------------------------------------- |
| **Risk**       | `react-native-android-widget` is less mature than native Glance; edge cases may arise. |
| **Likelihood** | Medium                                                                                 |
| **Impact**     | Low (widget is nice-to-have)                                                           |
| **Mitigation** | Keep the widget simple (display total savings + tap to open app).                      |

### R2: React Native learning curve

|                |                                                                                                                                                                             |
| -------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Risk**       | Despite FE skill transferability, we may struggle with RN-specific patterns (navigation, native modules, platform differences).                                             |
| **Likelihood** | Medium                                                                                                                                                                      |
| **Impact**     | Medium                                                                                                                                                                      |
| **Mitigation** | @vojtechsanda leads architecture, while in the beginning others take self-contained UI screens or components. Use Expo's managed workflow to minimize native-code exposure. |

### R3: Abandoning the existing Kotlin template

|                |                                                                                                                                                                                          |
| -------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Risk**       | Sunk cost — the existing template (architecture, auth, clean architecture) is thrown away.                                                                                               |
| **Likelihood** | Certain                                                                                                                                                                                  |
| **Impact**     | Low–Medium                                                                                                                                                                               |
| **Mitigation** | Expo + Supabase JS SDK makes auth and project scaffolding fast. The architectural patterns (clean architecture, separation of concerns) transfer conceptually even if the code does not. |

### R4: Performance for animations

|                |                                                                                                                                                                                      |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Risk**       | Complex animations could cause frame drops on lower-end Android devices.                                                                                                             |
| **Likelihood** | Low                                                                                                                                                                                  |
| **Impact**     | Medium                                                                                                                                                                               |
| **Mitigation** | Use `react-native-reanimated` (runs on UI thread, not JS thread). Use Lottie for pre-rendered animations instead of runtime-computed ones. Keep animations decorative, not blocking. |

## 5. Consequences

**Positive:**

- All team members productive from day one.
- iOS support available without additional development effort.
- Access to the largest cross-platform mobile ecosystem (libraries, tutorials, community).
- Supabase integration is straightforward with the mature JS SDK.

**Negative:**

- Existing Kotlin template investment is lost.
- Widget development is less flexible than native Android.
- React Native adds a JavaScript bridge layer — acceptable for this app's complexity.
